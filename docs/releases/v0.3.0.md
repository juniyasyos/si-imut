# Release v0.3.0

**Tanggal**: 2026-06-13

## Ringkasan

Rilis proof-of-concept **GraphRAG ringan** berbasis Python. Sistem ini memungkinkan
query knowledge base project berbasis dokumentasi `docs/` dengan keyword scoring
dan dukungan LLM opsional.

---

## Struktur Folder

```txt
rag/
├── input/              # Hasil copy dari docs/
├── output/             # Output chunks.json & graph.json
│   ├── chunks.json     # Chunk dokumen berbasis heading
│   └── graph.json      # Grafik pengetahuan (nodes + edges)
├── scripts/
│   ├── sync_docs.py    # Copy .md dari docs/ ke rag/input/
│   ├── ingest.py       # Chunking + graph extraction
│   └── query.py        # Query CLI
├── requirements.txt    # Python dependencies (ringan)
├── .env.example        # Template konfigurasi
└── README.md           # Dokumentasi GraphRAG
```

## Cara Menjalankan

### 1. Setup

```bash
cd rag
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Sync + Ingest

```bash
python3 rag/scripts/sync_docs.py
python3 rag/scripts/ingest.py
```

### 3. Query

```bash
python3 rag/scripts/query.py "jelaskan modul daily report"
```

## Contoh Query

**Query**: `jelaskan service SIIMUT`

**Output**:
```
============================================================
  Pertanyaan: jelaskan service SIIMUT
============================================================

  Jawaban:
  [Retrieval Only — no LLM configured]

  Chunk teratas yang relevan:
    [1] PROJECT_STRUCTURE.md → app/ — Source Code

  Sumber:
    • docs/PROJECT_STRUCTURE.md
```

**Query**: `command apa untuk migrate database?`

Dengan LLM dikonfigurasi, menampilkan jawaban lengkap dengan sumber.

## Output yang Dihasilkan

### chunks.json

Array chunk terstruktur dari dokumentasi. Setiap chunk memiliki:

- `id`: ID unik (`chunk-0001`)
- `source_file`: File asal (`PROJECT_STRUCTURE.md`)
- `heading`: Heading asal chunk
- `content`: Isi chunk

### graph.json

Grafik pengetahuan dengan:

- **Nodes**: Entity seperti Project (SIIMUT), Module (DailyReport), Service (IAM/SSO),
  Port, Container, Command, KnownIssue, Decision, Release
- **Edges**: Relasi seperti `filament --> uses --> iam-service`,
  `filament --> exposed_by --> nginx`, `decision-refactor --> affects --> siimut`

## Known Limitations

1. **Pattern-based extraction** — entitas hanya tertangkap jika kata kunci eksplisit
   muncul di dokumentasi.
2. **Source code tidak diindeks** — hanya markdown di `docs/`.
3. **Keyword scoring** — tanpa semantic search atau embedding.
4. **LLM opsional** — tanpa API key, hanya retrieval.
5. **No auto-rebuild** — perlu menjalankan sync_docs.py + ingest.py manual.
